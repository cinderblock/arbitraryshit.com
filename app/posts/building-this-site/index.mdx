I had a domain sitting around and a pile of little projects with nowhere to
live. So: a blog. This first post is mostly here to prove the plumbing works —
text, pictures, source code, and interactive bits.

## The stack

Nothing off the shelf. It's my own `ssg-base` template — React Router in SPA
mode with every route prerendered to static HTML at build time — plus an MDX
layer for posts. Bun runs the show, Cloudflare Pages builds and hosts it on
every push.

Each post is just a folder:

```
app/posts/building-this-site/
  index.mdx           — this file
  pipeline.svg        — images live next to the words
  arbitrary-number.tsx — interactive components too
```

The whole "post registry" is one glob import plus route loaders that read
frontmatter off the filesystem at build time:

```ts
// In a route loader — runs at build time, so metadata ships as prerendered
// data for that route only. The JS bundle stays the same size no matter how
// many posts pile up.
export function loader() {
  return { posts: listPosts() };
}

// Post bodies are only ever imported dynamically — each post (and any
// interactive components it drags along) is its own chunk.
const bodies = import.meta.glob<{ default: ComponentType }>(
  "../posts/*/index.mdx",
);
```

## The repo card

That box up top comes from two frontmatter lines: the repo name and the
commit this post was written against. A daily job asks GitHub how far the

card says the repo is fifty commits ahead, trust the repo, not me.

## Writing workflow

There is none, which is the point. Make a folder, write markdown, commit,
push. Cloudflare builds it. See you next week, probably.
