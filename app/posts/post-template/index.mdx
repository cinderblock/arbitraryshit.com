This post is a template (and a living test of the draft, cross-link, image,
and code-highlighting machinery — it links to the first post, which shows a
backlink here in dev).

## Getting started

To start a new post:

1. Copy this folder to `app/posts/<your-slug>/` — the folder name is the URL.
2. Fill in the frontmatter above. `title`, `date` (quoted!), and
   `description` are required. While `draft: true` is set, the post is
   listed in dev but unlisted in production (the URL still works as a
   noindexed preview). Delete the flag to publish.
3. Write. Markdown, code blocks, tables all work.

## Images and components

Images live in the folder and get imported:

```mdx
import photo from "./photo.jpg";

<img src={photo} alt="Describe it" />
```

Which renders like this:

<img src={example} alt="A placeholder example image colocated with the post" />

Interactive components too — any `.tsx` file in the folder:

```mdx
import { Widget } from "./widget";

<Widget />
```

## Linking posts

Three ways to link posts together:

- `builds-on: [slug]` — directional, for tools built on top of other tools.
  Shows "Builds on …" under this post's title and "Built on by …" at the
  bottom of the other post, so old posts point forward automatically.
- `related: [slug]` — non-directional. Also symmetric: listing a post here
  puts a backlink on it.
- Inline, mid-sentence: `[the LED driver](/posts/led-driver)` — plain
  markdown links to post URLs work anywhere in the body.

Slugs are checked at build time — a typo fails the build.
